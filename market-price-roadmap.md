# Market-Price 技能演进路线图

> 创建时间：2026-03-01
> 背景：采购物料价格与库存监控 Skill 头脑风暴

---

## 愿景

**一句话概括**：单品查价工具 → 全品类多渠道自动扫描监控系统 → 嵌入 ERP 的采购决策辅助引擎

**核心价值**：让采购价格透明化、决策数据化，用 AI + API 替代过去靠人工询价、凭经验判断的模式。

---

## 当前状态

- **MVP**：通过 Mouser API 抓取电子料的现货价格，与公司实际采购价做对比（经验比例约为网上批量价的 1/3），形成价格模拟曲线

- [x] Mouser API 接入

### v2.0 - 价格 + 库存双指标监控

**目标**：从单一价格 → 价格 + 库存双指标

| 指标 | 信号 | 行动 |
|------|------|------|
| 市场价格偏高（超出正负阈值） | 市场可能涨价 | 提前备货 / 锁定订单 |
| 市场价格偏低 | 公司采购价偏贵 | 找供应商谈降价 |
| 市场库存充足 | 不缺货 | 不急着囤货 |
| 市场库存稀少 | 可能缺货 | 提前备货 |

**阈值设计**：
- 按品类/价格区间分别设定
- 低价料：10%-20%
- 高价料：5% 左右
- 红绿黄灯可视化呈现

**多渠道交叉验证**：
- [x] Mouser（已跑通）
- [ ] 立创商城
- [ ] Digi-Key / Smith / 其他现货商

**判断逻辑**：采用"最大集"原则——只有三个渠道都无库存才判定缺货；只要一个有库存，就认为市场不缺。

---

### v3.0 - 全品类覆盖

**阶段性扩展路径**：

1. **电子料**（当前 MVP，用现货网站 API）
2. **PCB 原材料**（如 CCL 覆铜板等）
3. **大宗商品** — 铜价
   - 数据源：上海有色金属网 + 伦敦金属交易所（LME）
   - 优先用 API，不行就用爬虫
   - 两地价差理论上≈汇率，可做套利/趋势分析
4. **其他原材料**（如 TR 等品牌官网可直接抓价格）

---

### v4.0 - 定时自动扫描 + 趋势预测

**运行方式**：
- 利用 Cron 定时任务，自动执行扫描
- 建议频率：每周一次
- 流程：公司物料清单（品牌 + MPN）→ 循环调用 API 查价格/库存 → 与公司数据对比 → 输出异常报告

**趋势预测**：
- 2-3 个月数据即可形成价格波动曲线
- 结合 Python 预测库（如 Scikit-learn）做价格趋势预测
- 实现从"被动响应"到"主动预判"市场走向

---

### v5.0 - 嵌入 ERP，采购决策辅助

**最终愿景**：将 Skill 封装为 API 接口，直接嵌入公司 ERP 系统

- 采购审批页面旁边实时显示市场参考价
- 红绿黄灯自动标注价格合理性
- 审批人一眼就能判断"这个价格该不该批"

**解决核心痛点**：老板审批价格时缺乏市场参考，容易被采购"牵着鼻子走"
